scalaz.stream.mongodb

MongoRuntimeConfig

case class MongoRuntimeConfig(bindPort: Int = 27717, bindIp: String = "127.0.0.1", dataPath: Option[Path] = scala.None, nssSize: Int = 1, noPrealloc: Boolean = true, noJournal: Boolean = true, cmdLinePars: Option[String] = scala.None, mongodPath: Option[Path] = scala.None, mongodEchoStdOut: Option[OutputStream] = scala.None, mongodEchoStdErr: Option[OutputStream] = scala.None) extends MongoSpecificationConfig with Product with Serializable

Basic configuration class to fine-tune mongo runtime. If this configuration is used, then tests spawns own mongo instance and creates databases. Instances are spawn concurrently, so it is completely safe to use specifications that are run in parallel. Examples must however be written in a way that they do not use conflicting namespaces. Please note concurrent execution may require significant amount of RAM, make sure you have few GB spare on top of your test environment.

Databases are than cleaned after full spec is processed and instance of mongo is teared down.

bindPort

port to bind to. Default is 27717 to not clash with any local instance that may run. This is initial port to use specs takes track of used ports, and will increment this for each concurrent specs run. Once the db is shutdown, the port is released and will be reused by other database. If the Examples are run sequentially the spec guaranties the mongodb port will be always equal to this number (so you can sniff for example on this port)

bindIp

ip to bind to, usually default is just ok

dataPath

path where data for mongo instance are stored, leave empty for system to generate temp dir and clean up it when needed

nssSize

size of namespace file in MB. For testing we have decreased default to low value (1 MB).

noPrealloc

by default this is true, but hence we sparing resources in specs we set this to true

noJournal

by default this is true, and disables journalling. This speeds up the startup of database in certain situations

cmdLinePars

other command line parameters to pass. please see http://docs.mongodb.org/manual/reference/program/mongod/ for possibilities

mongodPath

To run mongod from specific location this has to be set. If not set it is resolved automatically via querying system variable SPEC_MONGO_HOME that should point to mongo installation dir (parent of bin/mongod)

mongodEchoStdOut

If specified, Echoes stdout output of mongod process to provided stream

mongodEchoStdErr

If specified, Echoes stderr output of mongod process to provided stream
